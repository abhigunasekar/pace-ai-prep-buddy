# Project Brief

## Context

PACE is a research and product prototype for HBS case learning, developed with Professor Malcolm Baker's instructor perspective central to the design.

The tension is structural:

- Students can use general AI tools to jump straight to finished case answers.
- Instructors want students to wrestle with concepts, assumptions, judgment, and model logic.
- Many MBA students need more scaffolding to build finance intuition and confidence.

PACE resolves that tension by making AI a learning scaffold instead of an answer machine.

## Product Thesis

If an AI tutor forces students to move through the PACE loop, it can improve preparation quality while preserving the learning value of the case method.

## User Segments

- Student: wants to prepare efficiently, understand finance concepts, build a model, and speak in class.
- Instructor: wants higher-quality preparation without outsourcing the intellectual work.
- Course team: wants reusable case flows, prompt guardrails, and evidence that the tool improves learning.

## Initial Scope

- Finance case preparation.
- Socratic tutoring and layered hints.
- Case-specific flows built from reusable templates.
- Evaluation framework focused on learning integrity.

## Out of Scope

- Finished case writeups.
- Fully generated Excel models.
- Recommendation memos written without student reasoning.
- Generic chatbot behavior that optimizes for fast answers.
